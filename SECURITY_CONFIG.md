# 智眠星运营终端 - 安全配置

## 登录安全策略

以下参数可在 `server/index.js` 中的 `SECURITY_CONFIG` 对象中配置：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `requireVerifyCodeAfterFailures` | 3 | 登录失败N次后需要验证码 |
| `lockThreshold` | 10 | 登录失败N次后锁定账户 |
| `lockDurationMs` | 1800000 (30分钟) | 账户锁定时长（毫秒） |
| `verifyCodeExpiryMs` | 300000 (5分钟) | 验证码过期时间（毫秒） |

## 当前配置

```javascript
const SECURITY_CONFIG = {
  // 登录失败3次后需要验证码
  requireVerifyCodeAfterFailures: 3,
  
  // 账户锁定阈值（错误次数上限）
  lockThreshold: 10,
  
  // 锁定时长（毫秒）- 默认30分钟
  lockDurationMs: 30 * 60 * 1000,
  
  // 验证码过期时间（毫秒）- 默认5分钟
  verifyCodeExpiryMs: 5 * 60 * 1000,
}
```

## 测试场景

1. **正常登录**: 输入正确手机号和密码
2. **失败3次**: 连续输入错误密码3次，第4次需要验证码
3. **失败10次**: 连续输入错误密码10次，账户锁定30分钟
4. **锁定后登录**: 锁定期间尝试登录，显示剩余解锁时间

## 生产环境建议

1. 将 `loginAttempts`、`verifyCodes`、`accountLocks` 改用 Redis 存储
2. 验证码应使用图形验证码（如 svg-captcha）
3. 添加登录日志记录
4. 添加异地登录检测
5. 添加短信验证码支持
